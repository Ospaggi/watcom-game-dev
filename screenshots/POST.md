# [자작] Watcom C++로 만든 486용 로그라이크 "던전 크롤러" (VGA + 애드립)

안녕하세요, QuickBASIC입니다.

작년부터 취미로 만들던 도스용 게임 엔진 위에 샘플 게임을 하나 완성해서 올려봅니다.
486에서 돌아가는 것을 목표로 만든 턴제 로그라이크 던전 크롤러입니다.

## 엔진 이야기

엔진은 Watcom C/C++ 10.6으로 만들었고, DOS4GW 익스텐더 기반 32비트 플랫 모델입니다.
재미있는 점은 빌드 방식인데, 맥에서 소스를 편집하고 컴파일은 DOSBox 안에서 실제
Watcom 툴체인(WCL386)으로 합니다. 당시 개발 환경 그대로인 셈입니다.

주요 구성은 이렇습니다.

- **그래픽**: VGA Mode 13h (320x200, 256색), 더블 버퍼 + vsync
- **컴파일드 스프라이트**: PNG 이미지를 x86 기계어(MOV 명령 스트림)로 변환해 두고,
  그릴 때는 `call` 한 번으로 실행합니다. 반복문/분기 없이 픽셀을 박아 넣기 때문에
  486에서도 그리기 속도가 아주 빠릅니다. 투명 픽셀은 아예 명령을 만들지 않습니다.
- **한글 출력**: 소스 코드에서 실제로 쓴 글자만 뽑아서 16x16 한글/8x16 영문 글리프를
  같은 방식(컴파일드 코드)으로 구워 FONT.BIN 하나에 담습니다.
- **타이머/입력**: PIT를 1000Hz로 재프로그래밍한 밀리초 타이머(INT 8), INT 9 후킹
  키보드 핸들러(실시간 키 상태 + 이벤트 링버퍼)
- **사운드**: 애드립(OPL2) VGM 음악 플레이어를 타이머 ISR에 체인해서 재생하고,
  동시에 사운드블라스터 PCM 효과음을 소프트웨어 믹서(auto-init DMA + IRQ)로 섞습니다.
  음악과 효과음이 같이 나옵니다.
- **에셋 파이프라인**: PNG/WAV → 게임 포맷 변환은 호스트 쪽에서 TypeScript(Bun)
  도구들이 처리합니다.

## 샘플 게임 "던전 크롤러"

엔진 검증용으로 만든 게임입니다. 원래 종스크롤 슈팅을 만들다가 486(DOSBox cycles
10000 기준)에서는 화면에 움직이는 것이 많으면 버거워서, 아예 "입력할 때만 화면을
다시 그리는" 턴제 로그라이크로 방향을 바꿨습니다. 덕분에 CPU가 느려도 조작감이
전혀 달라지지 않습니다.

- 절차 생성 던전(방 + 복도) — 들어갈 때마다 지형이 달라집니다
- 지하 1층부터 4층까지 내려가서 보스 드래곤을 잡으면 클리어
- 시야(FOV)는 재귀 섀도캐스팅으로 처리했습니다. 부동소수점 없이 정수 분수 연산만
  씁니다. 밝은 시야 바깥 2칸은 체커보드 디더로 "어스름하게" 보입니다.
- 전투는 로그라이크 전통의 범프 공격(몬스터 쪽으로 이동)
- 몬스터 9종: 쥐, 거미, 고블린, 스켈레톤, 오크, 슬라임, 다크나이트, 마법사, 드래곤
- 아이템: 물약(HP 회복), 검(공격력 +2), 금화
- 게임 음악은 전부 **도스박물관 Ospaggi님이 작곡하신 OPL2(애드립) VGM 트랙**입니다.
  메뉴, 지하 1~4층, 엔딩까지 총 6곡이 장면마다 바뀌며 흐릅니다. 애드립 특유의
  FM 사운드가 던전 분위기와 정말 잘 어울립니다. 공격/피격/아이템/계단/승리/패배
  효과음도 따로 나옵니다.
- 엔딩을 보면 스크롤 크레딧이 올라갑니다

**조작법**: 방향키 또는 WASD 이동·공격 / P 물약 / ESC 나가기
