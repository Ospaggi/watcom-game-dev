# 던전 크롤러 (가제 "DUNGEON") 설계 문서

- 날짜: 2026-07-02
- 대상: watcom-game-dev 엔진 기반 DOS 게임 (VGA Mode 13h, 486 DOSBox cycles=10000)

## 1. 개요

지하 4층 던전을 내려가 최하층의 보스를 처치하는 턴제 로그라이크.
게임 흐름: 메인메뉴 → 1층~4층 → 보스 처치 → 엔딩. 사망 시 게임오버 → 메뉴.

턴제이므로 키 입력이 있을 때만 화면을 다시 그린다. 486 cycles=10000 환경에서
성능 병목이 없도록 실시간 애니메이션·스크롤을 배제한다.

## 2. 화면 구성 (320×200, 16×16 타일)

- **좌측 뷰포트**: 15×11 타일 (240×176). 카메라가 플레이어를 따라감.
  맵 크기는 층당 약 40×30 타일.
- **우측 패널 (80px)**: 층수, HP 바, 공격력, 소지 물약 수, 금화.
- **하단 (24px)**: 한글 메시지 로그 1줄. 기존 FONT 모듈(한글 16×16) 사용.

## 3. 핵심 시스템

### 3.1 맵 생성 (런타임 절차 생성)
- 방+복도 방식: 랜덤 방 배치(겹침 회피) → 복도로 순차 연결.
- 층마다 새로 생성. 내려가는 계단(`>`)에 도달하면 다음 층.
- 4층에는 계단 대신 보스 방 배치.

### 3.2 시야 (FOV)
- 시야 반경 5. 벽 차단은 Bresenham 직선(LOS) 체크.
- 탐험한 지역은 기억(fog of war): 어둡게/외곽만 표시.

### 3.3 전투 (범프)
- 적이 있는 칸으로 이동 입력 → 공격. 데미지 = 공격력 ± 소폭 랜덤.
- 플레이어 턴 후 모든 몬스터가 1턴 행동.

### 3.4 몬스터 AI
- 플레이어가 LOS 안에 있으면 한 칸 추적(greedy step), 아니면 랜덤 배회.
- 층별 2~3종 + 4층 보스. 층이 깊을수록 HP/공격력 상승.

### 3.5 아이템 (라이트 RPG)
- 물약: 줍고 P키로 사용, HP 회복.
- 무기 강화: 밟으면 공격력 영구 +.
- 금화: 점수.
- 경험치/레벨/장비 슬롯 없음. 성장은 아이템으로만.

## 4. 리소스 (웹 다운로드)

- **타일/스프라이트**: OpenGameArt·itch.io·craftpix 무료 16×16 던전 타일셋
  (CC0 우선, 후보: 0x72 Dungeon Tileset, Kenney). `mksprite --bin --grid`로 SPR 변환.
- **음악**: 기존 ST00~ST07.VGM 재활용 (층별 다른 곡, 메뉴/엔딩 곡 포함).
- **효과음**: OpenGameArt 무료 WAV (타격/물약/계단/사망) → `mksfx`로 SFX 변환.
- 라이선스는 CC0 또는 상업적 이용 가능만 사용, 출처를 문서에 기록.

## 5. 모듈 구성 (SRC/)

| 파일 | 역할 |
|------|------|
| `GAME.CPP` | 상태 머신(메뉴/플레이/엔딩/게임오버) + 메인 루프 (기존 스모크 테스트 교체) |
| `DUNGEON.CPP/H` | 절차 맵 생성, 타일 데이터, FOV |
| `MOB.CPP/H` | 플레이어/몬스터 상태, AI, 범프 전투 |
| `HUD.CPP/H` | 우측 패널 + 메시지 로그 |

기존 엔진 모듈(GFX/INPUT/TIMER/SOUND/SFX/SPRITE/FONT/IMG)은 수정 없이 사용.
init 순서 `timer_init()` → `input_init()` → `snd_init()`, shutdown은 역순(LIFO) 준수.

## 6. Watcom 제약 준수

- STL/예외/RTTI 금지, C 표준 라이브러리만.
- 파일명 8.3 대문자. 랜덤은 `rand()` (시드는 `timer_ms()`).
- 턴제라 ISR 추가 없음 — 기존 타이머/사운드 체인 그대로.

## 7. 검증

- 자동 테스트 없음. `./build.sh` → `./run.sh`로 육안 검증.
- 각 단계 완료 시 스크린 캡처(SCRNCAP)로 화면 확인.
- 사운드 변경 시 `./sfxtest.sh` 회귀 확인.
