# ASSETS

This project now uses procedural wireframe art instead of downloaded image packs.

## Layout

```text
ASSETS/
├── generated/          # PNG sheets generated by tools/draw_assets.ts
├── raw/audio_vgm/      # User-provided OPL2 VGM music
└── README.md
```

Downloaded sprite/background image packs were removed. Do not add web-downloaded
PNG/ZIP/RAR art back into `ASSETS/raw/`; keep the visual style reproducible from
code.

## Regeneration

Run:

```bash
./convert.sh
```

The pipeline:

1. `tools/mkpalette.ts` generates `SRC/PALETTE.H` and `tools/palette.json`.
2. `tools/draw_assets.ts` draws black-background wireframe PNG sheets into
   `ASSETS/generated/`.
3. `tools/mksprite.ts` converts generated sheets into `SRC/SPR_*.SPR` and
   `SRC/SPR_*.RAW`.
4. VGM files from `ASSETS/raw/audio_vgm/` are copied into `SRC/`.

## Runtime Assets

The game loads the generated DOS 8.3 runtime files from `SRC/`, including:

- `SPR_PSHP.SPR` for the player.
- `SPR_E*.SPR` for enemies and enemy bullets.
- `SPR_B*.SPR` for bosses.
- `SPR_EXPL.SPR` and `SPR_PWUP.SPR` for effects/items.

`SPR_EBLU.SPR` is the blue enemy. `SPR_EBL.SPR` remains the line bullet, avoiding
the old filename collision between the enemy and bullet assets. Stage backgrounds
are now rendered directly in `SRC/BG.CPP` as scrolling square grids with
stage-specific colors.
